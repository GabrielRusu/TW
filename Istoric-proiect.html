Introducere

In cadrul proiectului NEDA, ne-am propunes sa centralizam datele referitoare la cutremurul din Nepal, sa le prelucram intr-un mod structurat si coeerent si sa dezvoltam o aplicatie prin care se poate face accesarea lor intr-o maniera mult mai facila si mai variata spre a folosi publicului larg care este interesat de acest eveniment.

Autorii proiectului

3 studenti in anul 2, la Facultate de Informatica Iasi.

Scopul proiectului

Aceasta lucrare a fost realizata din mai multe motive. In primul rand, deoarece reprezinta o oprtunitate de a invata procesul de dezvoltare al une aplicatii web, atat pe partea de front-end cat si pe cea de back-end. In al doilea rand, deoarece avem sansa sa construim ceva folositor, care are potentialul de a deveni o contributie utila societatii. In al treilea rand, acest proiect reprezinta criteriul pentru notarea studentilor. See: Aspecte privind evluarea.

Tehnologii & limbaje de programre utilizate

Pentru a ne putea duce la bun sfarsit proiectul, am utilizat o serie de tehnologii software.

Front-end

Spring

framework in java pentru a ne usura munca in a crea si organiza codul in html si css.

Back-end

Tomcat

L-am folosi pentru a crea un server exclusiv in Java

Maven

Pe parte de server ne-a simplificat munca.

Oracle Database

Am ales aceasta baza de date deoarece era compatibila si cu tematica seminarului de PSGBD.

Java

Un limbaj versatil, pe care l-am studiat si la Programare Avansata.

Dificultatile de ordin tehnic pe care le-am avut

Vom mentiona fiecare din experienta sa situatii mai dificile dpv tehnic.

Gabi

greseli idioate

Stefan

In primul rand, cea mai mare problema a mea a fost lipsa totala de cunostinte despre spring sau despre orice altceva, a durat ceva pana mi-am dat seama cum se fac toate configurarile si pana sa invat cum se lucreaza cu acest framework.

Sanziana

Mi-am dat seama ca lipsa unei baze stabile de cunostinte de programare duce la repercusiuni. Faptul ca inca schiopatez la Java si SQL a facut intelegerea si aplicarea tutorialelor sau altor materiale explictive de pe internet mult mai dificila, fiind nevoita sa ma opresc frecvent, intalnind concepte necunoscute si facnd paranteze de recuperare a unei materii pe care nu am invatat-o la timp.

Un alt element care mi s-a parut dificil a fost utilizarea Oracle pentru bazele de date, fiind motivati de avantajele oferite seminarului de PSGBD. Cu MySQl sau SQL Server ar fi fost mai usor si mai placut de lucrat.

Conclusion

Lucrul la acest proiect ne-a deschis noi perspective, familiarizandu-ne cu indatoririle profesionale ale unui full-stack developer. Dincolo de cunostinte tehnice, am practicat si lucrul in echipa, descoperind ca este mult mai dificil decat ar fi parut la prima vedere, mai ales cand vine vorba de organizare si de mobilizare.
